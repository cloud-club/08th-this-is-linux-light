## [리눅스 사용자 및 그룹의 개념]
* 리눅스는 다중 사용자 시스템으로, 한 대의 컴퓨터에 여러 명이 접속하여 사용
* **루트(root)** 사용자는 가장 높은 권한을 가진 슈퍼 유저이며, 모든 작업 수행 가능
* 다른 사용자들은 하나 이상의 그룹에 소속되며, **사용자는 직원, 그룹은 부서**로 비유할 수 있음
* 사용자 정보는 `/etc/passwd` 파일에 정의되어 있음
    * `사용자 이름:암호(X):사용자 ID:그룹 ID:전체 이름:홈 디렉터리:기본 쉘` 순으로 구성됨
* 사용자 비밀번호는 `/etc/shadow` 파일에 암호화되어 저장됨
* 그룹 정보는 `/etc/group` 파일에 정의되며, 그룹 이름, 그룹 ID 등의 정보 포함

## [사용자 및 그룹 관련 명령어]
* `useradd ${유저명}`
    * *// 새로운 사용자를 추가하며, 옵션(-u: ID, -g: 그룹, -d: 홈 디렉터리, -s: 쉘)을 통해 세부 설정 가능*
    * *// 만약 그룹명을 명시하지 않는다면 유저명 그대로의 그룹이 생성되고 그 그룹에 유저 추가*
* `passwd ${유저명}`
    * *// 사용자의 비밀번호 지정*
* `usermod`
    * *// 사용자 속성 변경*
* `userdel ${유저명}`
    * *// 사용자 삭제*
* `chage`
    * *// 사용자 암호를 주기적으로 변경하도록 설정 가능*
* `groups`
    * *// 현재 사용자가 소속된 그룹 출력*
* `groupadd`
    * *// 새로운 그룹 생성*
* `groupmod`
    * *// 그룹 속성 변경*
* `groupdel`
    * *// 그룹을 삭제하지만, 사용자가 소속된 경우 삭제할 수 없음*
* `gpasswd`
    * *// 그룹의 암호를 설정하나, 일반적으로 잘 사용하지 않음*

## [파일과 디렉터리의 소유와 허가권]
* **첫 번째 자리** : 파일 유형
    * *// 하이픈(`-`) : 일반 파일 & `d` : 디렉터리*
* **두 번째부터의 9자리** : 파일 허가권
    * *// 3자리씩 끊어 읽음 (소유자, 그룹, 기타 사용자 순)*
* 그 이후부터 `링크 수`, `파일 소유자`, `파일 소유그룹`, `파일 크기`, `마지막 변경 날짜`, `파일 이름` 순으로 구성
* 허가권 각 자리는 `r`(읽기), `w`(쓰기), `x`(실행) 권한을 나타내며, 하이픈(`-`)은 권한 없음을 의미
* 허가권은 8진수로 표현 가능하며, `rwx`는 `421`의 가중치를 가짐 (예: `rw-r--r--`는 `644`)

#### 허가권 변경 (chmod)
* `chmod` 명령어를 통해 파일 허가권 변경 가능
    * ex. `chmod 777 파일명`은 소유자, 그룹, 기타 사용자 모두에게 모든 권한(읽기, 쓰기, 실행)을 부여

#### 소유권 변경 (chown, chgrp)
* `chown user.group file`은 파일의 소유 사용자와 그룹을 변경
* `chown`과 `chgrp`는 따로 사용할 수도 있음

## [링크]
* 리눅스에서는 **하드 링크**와 **심볼릭 링크**로 나뉨
    * **심볼릭 링크** : 윈도우즈의 '바로가기'와 비슷
    * **하드 링크** : 개념 상이

### `<하드 링크와 심볼릭 링크의 작동 방식>`
* 원본 파일은 **inode 블록**(파일 정보)과 **Data 블록**(실제 내용)에 저장됨
* **하드 링크** 파일은 원본 파일과 같은 **inode를 공유**
* **심볼릭 링크**는 새로운 inode를 생성하고, 원본 파일을 가리키는 **포인터(주소)를 저장**

### `<하드 링크 및 심볼릭 링크 생성 명령어>`
* `ln 링크대상파일이름 링크파일이름`
    * *// 하드 링크를 생성. 원본과 같은 inode 사용*
* `ln -s 링크대상파일이름 링크파일이름`
    * *// 심볼릭 링크를 생성. 새로운 inode를 만들고 원본 파일을 가리킴*

## [RPM (Red Hat Package Manager)]
* 윈도우즈의 `setup.exe`처럼 리눅스에서 프로그램을 설치/관리하는 파일 형식. 패키지라고 부름.
* RPM 패키지 파일명 : `패키지 이름-버전-릴리즈.리눅스버전.아키텍처.rpm` 순으로 구성

### `<RPM 명령어 사용법 (설치, 삭제, 조회)>`
* **패키지 설치**
    * `rpm -Uvh 패키지파일이름.rpm`
    * *// U: 업그레이드 또는 설치, v: 설치 과정 확인, h: 진행 과정을 `#`로 표시*
* **패키지 삭제**
    * `rpm -e 패키지이름`
* **설치된 패키지 확인**
    * `rpm -qa 패키지이름`
* **파일이 소속된 패키지 확인**
    * `rpm -qf /파일/절대/경로`
* **설치 전 RPM 파일 내부 목록 확인**
    * `rpm -qlp 패키지파일이름.rpm`
* **설치 전 상세 정보 확인**
    * `rpm -qip 패키지파일이름.rpm`

### `<RPM의 단점: 의존성 문제>`
* 특정 패키지 설치 시 필요한 다른 패키지(의존성)를 수동으로 먼저 설치해야 하는 번거로움이 있음
* 이러한 단점을 해결하기 위해 **DNF** 또는 **YUM** 명령어가 등장

## [DNF (Dandified YUM)]
* RPM의 의존성 문제를 해결하며, 인터넷 저장소를 통해 필요한 패키지를 자동으로 다운로드 및 설치
* RHEL 8부터는 기존 YUM을 개선한 DNF 명령어를 사용
* 패키지 저장소 정보는 `/etc/yum.repos.d/` 디렉터리 내의 파일(`*.repo`)을 통해 관리됨

### `<DNF 기본 사용법 및 주요 명령어>`
* `dnf -y install 패키지이름`
    * *// DVD 없이 인터넷을 통해 자동 설치(가장 많이 사용)*
* `dnf install RPM파일이름`
    * *// 다운로드받은 RPM 파일도 의존성을 해결하며 설치 가능*
* `dnf check-update` : 업데이트 목록 확인
* `dnf update` : 패키지 업데이트
* `dnf remove` : 패키지 삭제
* `dnf info` : 패키지 정보 확인

### `<DNF 고급 사용법>`
* **패키지 그룹 설치**
    * `dnf groupinstall "패키지 그룹 이름"`
* **패키지 리스트 확인**
    * `dnf list 패키지이름`
* **특정 파일이 속한 패키지 이름 확인**
    * `dnf provides 파일이름`
* **GPG 키 검사 생략**
    * `dnf install --nogpgcheck rpm파일이름.rpm`
* **기존 저장소 목록 제거**
    * `dnf clean all`

### `<DNF 작동 방식 이해>`
* `dnf install` 명령을 실행하면 `/etc/yum.repos.d/*.repo` 파일에서 **저장소 주소**를 확인
* 해당 주소에서 설치 가능한 패키지 목록을 가져온 후, 필요한 패키지와 의존성 패키지를 모두 다운로드하여 설치
* `-y` 옵션을 추가하면 모든 과정을 자동으로 한 번에 처리

## [파일의 압축과 검색]

### 파일 압축
* 리눅스 파일 압축 확장자는 `xz`, `bz2`, `gz`, `zip` 등이 있으며, `xz`와 `bz2`의 압축률이 우수
* `xz 파일명` : .xz로 압축 (원본 파일 사라짐)
    * `xz -d 파일명.xz` : 압축 해제
* `bzip2 파일명` : .bz2로 압축 (원본 파일 사라짐)
    * `bzip2 -d 파일명.bz2` 또는 `bunzip2 파일명.bz2` : 압축 해제
* `gzip 파일명` : .gz으로 압축 (원본 파일 사라짐)
    * `gzip -d 파일명.gz` 또는 `gunzip 파일명.gz` : 압축 해제

### 파일 묶기 (tar)
*// 리눅스에서는 파일 압축과 묶기가 별개의 기능*
* `tar [동작][옵션] 파일명 대상`
    * **동작**: `c`(묶기), `x`(풀기), `t`(내용 확인)
    * **옵션**: `f`(파일 지정), `v`(과정 보이기), `J`(.xz), `z`(.gz), `j`(.bz2)
* **tar 사용 예시**
    * `tar cvf my.tar /etc/sysconfig` : 묶기
    * `tar cvfJ my.tar.xz /etc/sysconfig` : 묶고 .xz로 압축하기
    * `tar xvf my.tar` : 묶음 풀기
    * `tar xvfJ my.tar.xz` : .xz 압축 풀고 묶음 풀기

### `<파일 위치 검색>`
* `find [경로] [옵션] [조건]` : 가장 강력하고 기본적인 파일 검색 명령어
    * **옵션**: `-name`, `-user`, `-newer`, `-perm`, `-size` 등
    * **Action**: `-print`(기본값), `-exec 외부명령 {} \;`(검색된 파일에 명령 실행)
* **find 사용 예시**
    * `find /etc -name "*.conf"`
    * `find /home -name "*.swp" -exec rm {} \;`
* **그 외 검색 명령어**
    * `which 실행파일이름` : PATH에 설정된 디렉터리에서만 실행 파일 검색
    * `whereis 실행파일이름` : 실행 파일/소스/man 페이지까지 검색
    * `locate 파일이름` : 파일 목록 데이터베이스에서 검색 (매우 빠름)
