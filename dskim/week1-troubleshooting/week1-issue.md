2. 현상 : Rocky Linux 9.0을 통한 VM 생성 시 해당 화면에서 정지됨
![이슈1](./img/이슈1.png)

행동 :
- 혹시 메모리 문제인가 싶어서 메모리를 2GB -> 4GB로 확장했으나 동일한 문제 발생
- 강의에서 말했던 필요없다고 삭제한 다른 Device들 때문인지 싶어 아무것도 건들이지 않고 그대로 진행했으나 여전히 동일한 문제 발생
- 이번엔 한번에 4개의 VM을 진행해서 그런가 싶어 "Server" VM만 홀로 진행했으나 동일한 문제 발생

해결 : Rocky Linux 9.0은 초기 버전이라 M1과 같은 ARM 아키텍처는 지원이 부족할 수도 있다고 해서 최신 버전인 9.6으로 해봤더니 드디어 생성은 되었으나 문제는 강의와 많이 달라지기 시작

3. 현상 : 강의에서 시키는대로 NVMe 하드디스크를 삭제하고 SCSI 하드디스크를 생성했으나 로컬 표준 디스크에 탐색이 되지 않음
![이슈2](./img/이슈2.png)

행동 :
확실하지는 않으나 과거 Intel 기반 Mac의 VMware에서는 SCSI 방식이 널리 사용되었지만, M1/M2와 같은 Apple Silicon 환경에서는 가상 디스크의 표준 방식으로 NVMe를 사용하니 이로 인해 호환이 되지 않아 탐색이 되지 않는다고 추측

해결 : 강의에서 말하는 것처럼 SCSI가 아닌 기존에 기본값으로 있던 NVMe로 복구했더니 탐색이 됨
